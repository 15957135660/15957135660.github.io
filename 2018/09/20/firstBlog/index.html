how to start a personal blog

前言：一直想自己搭建一个自己的博客，观感里，觉得这是比较麻烦的事，包括搭建服务器，申请域名等等工作。偶然发现搭建自己的博客并不困难，于是，基于网上的教程，开始了自己博客的搭建。

​ 搭建注意事项：

1.参考文章每一步，搭建基本没有大问题，在第6步 Hexo 与 GitHub page 联系起来 时，文章内容有笔误

#分享# 自己搭建免费博客 Hexo + Github 以及 Coding.net

​ 原创： 星辰 最亮的北极星 ;) 4天前

​ 一路的坑，终于挺过来了😂，现在分享一下，有兴趣的可以尝试一下呦。

首先是搭建在 Github上

*一、 Hexo + Github*

1.首先安装 Node.js

百度一下，上官网下载相应的版本

安装教程：https://www.runoob.com/nodejs/nodejs-install-setup.html（PS:里面步骤五选择默认就好），别忘了按照教程检测 PATH 环境变量是否配置了 Node.js

2.安装 Git，同样官网下，这个直接下有点慢，我用了迅雷就快多了😂。

安装教程：https://jingyan.baidu.com/article/9f7e7ec0b17cac6f2815548d.html（PS：第四步我直接全选了），重点还是检查环境变量（就是第八步）。

3.接下来，去 Github 建立项目，项目必须要遵守格式：账户名.github.io，不然接下来会有很多麻烦。并且需要勾选 Initialize this repository with a README。（我的是已经存在了，所以红色提示）

4.建完项目后，然后点击 Settings，

然后下拉找到 GitHub Pages，你会看到有个提示的网址，没错，那就是你的博客部署页面，兴奋不。

5.接下来，安装 Hexo ，自己找个合适的地方创建个文件夹，我实在F盘创建了一个 MyBlog，然后用 CMD 进入到该文件夹下，例如我的，

切换到目录下后，然后输入 npm install hexo -g ，回车，开始安装，需要等待一小会，安装之后，输入 hexo -v，检查安装是否成功

输入 hexo init，初始化该文件夹，等待呀等待

初始化完成后，再输入 npm install 安装所需要的组件。

\6. CMD 不要关闭，最小化就好，接下来将 Hexo 与 GitHub page 联系起来。

首先设置 Git 的 user name 和 email name 。

​ 在你建立的文件夹下空白处右击，点击 Get Bash Here。然后在弹出的框内输入 git config –global user.name “Github的用户名”，然后回车，再输入

git config –global user.name “Githhub绑定的邮箱”，回车。接着，再输入 ssh-keygen -t rsa -C “Github绑定的邮箱”，连续三个回车，生成密钥，密钥文件在

C:\用户\ Administrator 或你的当前用户名.ssh

接着输入 eval “$(ssh-agent -s)”，添加密匙到 ssh-agent

再输入 ssh-add ~/.ssh/id_rsa，添加生成 SSH key 到 ssh-agent ，Git Bash先不要关，最小化即可，现在先打开上图中的第二个文件（记事本的方式打开），crtl + A 全部复制下来，接下来要用。

登录 Github，点击头像，再点击 settings，然后点击左侧列表ssh，如下图

点击右侧的 new ，建立新的 key。

Title 随便输入，key 则将我们之前复制的内容粘贴过来，然后 add 即可。

回到 Git Bash，我们来测试一下，输入 ssh -T git@github.com，测试添加ssh是否成功

如果你看到 Hi 后面是你的用户名，那么恭喜你，成功了，加入配置失败了，那么只要以下步骤就能完全解决：首先，清除所有的key-pair

输入ssh-add -D，回车，再输入rm -r ~/.ssh 删除你在github中添加的 key

重新生成 ssh 密钥对

ssh-keygen -t rsa -C “邮箱”

接下来正常操作，按照之前步骤，复制key，然后在在 github 上添加 key

最后再输入 ssh -T git@github.com 测试一下

7.在你建立的文件夹下找到 _config.yml文件，用 Sublim text（没有就用记事本）打开，翻到最后，接下来重点来了。首先看我的配置，

你要是只在 Github上搭建博客，那最下面coding那行不要就行，其余的照着我的该就行，xingchen007是我的用户名，修改成你自己的。（PS:y一定要注意缩进！！！一定要注意缩进！！！）

8.新建一篇博客，在 cmd 中输入 hexo new post 名称，

这时候你会在 你的文件夹\source_posts 目录下看到新建立的 md 文件，这就是你的新博客，编辑 md 文件推荐编辑器 Typora（具体网上搜了）。

9.最后就是部署了（激不激动!），在 cmd 中输入 npm install hexo-deployer-git –save 安装一个扩展，使用编辑器编辑好你的文章后那么就可以布署了，输入 hexo d -g，接下来就等部署完成了，中间出现很多东东都不用管，

部署成功后访问 http://用户名.github.io，那么你将看到你部署的博客了，我今天部署的如下（PS：如果 Coding.net 上搭建博客不需要的话那就直接翻到文章末尾，有注意事项）

*二、 Hexo + coding.net*

​ 用 Hexo + Github 搭建好博客之后，我又在想 Coding.net，之前张哥推荐过，因为在 部署在Github上 访问比较慢，所以我想着能不能在 Coding.net 搭建一个，于是上网搜了教程，一看，好简单，心一横!，也搭建一下，同时部署在两个平台上（反正都不要钱!）。

1.首先是文件夹下_config.yml文件，对它进行配置，在之前搭建的下面加一行就中，如下

在 Coding.net 上新建项目

项目名称格式 用户名 + .coding.me，其余默认就好，进入项目点击左边列表的代码，然后点击 Pages 服务 ，拉到最底下，选择一键部署，部署之后如下图，

访问地址就是你的博客地址，然后点击右边的设置（小轮子）

部署的分支选择 master，如果自动选择了，那就不用动了，接着就是 ssh（熟悉不？），点击项目列表的设置

然选择最下方的部署公钥，点击新建部署公钥，

名字随意，内容和 Github 那个一样。OK，在 Coding.net 上的就搭建完了，之后更新部署的时候， 会同时更新 Github 和 Coding 上的，哈哈，完美。试了试，果然还是国内快些···

（PS: 每次更新部署需要在 cmd 上，首先 cd 到你那个文件夹，然后输入更新部署的指令 hexo d -g，然后等待更新部署完就可以访问查看最新的了，建立文件则是 hexo new post 名称，记住，你的博客在 source_posts下）
